# DECISIONS

**CASE_ID:** resolved at runtime from the `CASE_ID` env var (placeholder `CEDX-7F3A`
during development). Role R and threshold T are computed from `sha256(CASE_ID)` — see §Amendment.

## What I deliberately did NOT automate (and why)
- **Final human approval.** The operator `approve/reject/request-changes/edit-resolve`
  action is a human decision by design. The pipeline drafts + verifies, but delivery is
  gated on a recorded human approval. Automating it away would defeat the governance the
  task is testing.
- **The CASE_ID amendment second approval.** High-value records (`amount >= T`) require a
  second, *different* approver in role R (maker ≠ checker). We enforce the gate in code but
  never auto-satisfy both approvals with one actor.
- **PDF OCR / arbitrary layouts.** Intake handles the seed's text-based PDFs (pypdf +
  an ASCII85/Flate fallback). Scanned-image OCR is out of scope; such a record would fail
  parsing and route to the exception queue rather than be silently dropped.

## Outlier threshold (and why it generalizes)
Robust z-score via **median + MAD** (`cedx/normalize/detectors.py`):
`robust_z = 0.6745 * (x - median) / MAD`, flag when `|robust_z| > 3.5`.
- Median and MAD are computed **per batch** and are themselves robust to the outlier, so a
  single huge value (seed REC-013 = 250000) does not inflate the scale the way mean+stdev
  would. On the held-out seed (different magnitudes, shuffled order) the threshold adapts to
  that batch's own spread — **no hardcoded constant anywhere**.
- `3.5` is the standard robust-z cutoff (~0.05% false-positive rate under normality).

## Abstain threshold
The Worker sets `abstain=true` (confidence < 0.5) when the source is too ambiguous to
produce a source-grounded draft (seed REC-021 unknown category, REC-015 contradictory body).
Abstain routes to `LOW_CONFIDENCE` — we never guess.

## Router policy + cost numbers
Default to the **cheap** model; escalate to **strong** only on (a) a retry, (b) a
verifier-flagged record, or (c) an unusually large amount. Justification: the vast majority
of records are trivial, so paying strong-model prices by default is pure waste.
- Measured on the dev seed (offline replay): **total $0.00085**, **avg $0.0000499/record**,
  **p95 latency 0.012 ms** (replay path; real-model latency dominates on the keyed path).
- **Projected at 10,000 records/day: ~$0.50/day** at the cheap-model price for two calls
  (worker + verifier) per clean record. Escalation only fires on hard records, so the
  projection is an upper bound for a clean batch.
- Per-record ceiling `MAX_COST_USD_PER_RECORD` (default 0.05) + `MAX_STEPS_PER_RECORD`
  (default 6); breaching either raises `BUDGET_EXCEEDED` → route, never silent overspend.

## How provenance survives re-run
- Transcripts are content-addressed: `transcripts/<response_hash_hex>.json`. Re-running
  in replay mode reads the same files and produces byte-identical hashes.
- The event log is **append-only and hash-chained** (`cedx/audit/log.py`): each event hashes
  in the previous event's hash. `verify_chain()` detects any mutation or deletion of a past
  entry (`make probe-append-only`).
- `make demo` rebuilds `out/` deterministically from the seed, so re-running never
  double-delivers (`make probe-idempotency`).

## What breaks first at 10k records/day
- **Serial processing.** The current run is single-process and serial. At 10k/day the first
  bottleneck is throughput, not cost — the fix is to shard records across workers (the agents
  are stateless per record, so this parallelizes cleanly).
- **Verifier as second LLM call** doubles per-record LLM spend; at scale I'd add a
  confidence gate to skip the Verifier on the highest-confidence, lowest-value records.

## Real-vs-faked (honesty)
- The **real graded path** (`REPLAY_LLM=false` + a key) makes the LLM load-bearing: Worker
  and Verifier call an OpenAI-compatible model via `cedx/llm/client.py`.
- The **offline path** replays committed transcripts. Those transcripts were generated by a
  deterministic dev generator (`ALLOW_FAKE_LLM`, gated so it can never stand in for a real
  model during grading). Everything except the raw model text is real, executing code.
